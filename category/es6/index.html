    # 说说let、const、var的区别

    1. let用于定义变量，存在TDZ、块级作用域、不存在变量声明提升、不可以重复声明变量
    2. const用于定义常量，常量的值不可变，一旦创建必须立即赋值、其余与let一致
    3. var声明的变量会挂载到window上，存在变量声明提升、可以重复声明变量

    # 注意

    1. const实际保证的并非值不可变，而是变量指向的那个内存地址保存的数据不可变
    2. 对于基本类型的数据，值就保存在变量指向的内存地址中，形如常量
    3. 对于引用类型的数据，变量指向的内存地址，保存的是指向实际数据的指针，const仅保证指针是固定的，但不能确保变量的结构不变

    # 对象的遍历

    1. for in(遍历自身属性和继承的可枚举属性，不含symbol)
    2. Object.key(obj)返回一个数组，包含了对象自身所有可枚举属性(不含继承)的key
    3. Object.getOwnPropertyName(obj)返回一个数组，包含对象自身的所有属性(包含不可枚举属性)的key
    4. Reflect.ownKeys(obj)返回一个数组，包含对象自身所有key(不含继承)，无论key是Symbol还是string，也不管是否可枚举

    # 新API

    # Object.isopen in new window(val1, val2)

    严格判断两个值是否相等

    注意

    +0 === -0   // true
NaN === NaN // false

Object.is(+0, -0)   // false
Object.is(NaN, NaN) // true

    # Object.assign(target, obj2, obj3...)

    合并对象，将其它对象的所有可枚举属性合并到目标对象上

    注意

    • Object.assign是浅拷贝，遇到同名属性时后者会覆盖前者

    # Object.setPrototypeOf(obj, prototype)、Object.getPrototypeOf(obj)

    修改对象的原型对象、获取对象的原型对象

    # Object.fromEntries()

    将键值对数组转为对象

    // { name: 'a', age: 19 }
Object.fromEntries([
    ['name': 'a'],

    # Set

    类似于数组，但成员的值都是唯一的

    # 基本操作

    • add(value)
      • 可链式调用
    • delete(value)
    • has(value)
    • clear()
    • size
    • keys()、values()、entries()、forEach(value, key)
      • 前三种遍历方式返回的都是迭代器对象
      • 由于Set没有键名，只有键值，因此keys()、values()返回结果一致
      • values()可以简化成for of

    # 实用场景

    • 数组、字符串去重
    const uniqueArr = arr => [...new Set(arr)]
const uniqueStr = str => [...new Set('abbac')].join('')

    # Map

    类似于对象，本质是键值对的集合(Hash结构)，键不限制范围

    # 基本操作

    • set(k, v)
      • k重复，会覆盖v，可链式调用
    • get(k)
    • has(k)
    • delete(k)
    • clear()
    • size
    • 遍历同Set

    # 结构转换

    # Map转对象

    function mapToObj(mapper) {
    let obj = Object.create(null)
    for (let [k, v] of mapper) {
        obj[k] = v
    }
    return obj
}

    # 对象转Map

    // planA
function objToMap(obj) {
    let mp = new Map()
    for (let k of Object.keys(obj)) {
        mp.set(k, obj[k])
    }
    return mp
}

// planB
function objToMap(obj) {
    return new Map(Object.entries(obj))
}

    # Proxy

    # 概述

    用于修改某些属性的默认操作，属于元编程，Proxy会在操作目标对象前进行拦截

    # 基本使用

    • new Proxy(target: object, handler: object)
    • get(target, propKey, receiver: Proxy实例本身)
    • set(target, propKey, value, receiver)
    • has(target, key)
      • 用于拦截hasProperty()，隐藏某些属性，不被in运算符发现
    • deleteProperty(target, key)

    # Reflect

    • Reflect与Proxy的方法一一对应

    示例

    // Proxy拦截obj对象的setter，在确保完成原有行为后，再添加额外操作
let proxy = new Proxy(obj, {
    set(target, propKey, value, receiver) {
        let success = Reflect.set(...arguments)
        if (success) {
            ...
        }
        return success 
    }
})

    # 实现简易的观察者模式

    const observerList = new Set()
// 添加观察者
const observe = fn => observerList.add(fn)

const observable = obj => new Proxy(obj, {set})

function set(target, propKey, value, receiver) {
    const result = Reflect.set(...arguments)
    observerList.forEach(observer => observer())
    return result
}

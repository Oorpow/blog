    # 1. 什么是Vue？

    用于构建用户界面的渐进式JS框架

    # 2. Vue的核心是什么？

    1. 数据驱动(MVVM)
    2. 组件化

    # 3. 什么是组件化？

    1. 每一个.vue文件均可以视为组件
    2. 相同逻辑、相同UI都可以抽离成组件

    # 3.1 组件化的优势

    1. 降低系统耦合度
    2. 提高维护性

    # 4. Vue与React的浅层对比

    1. diff算法(React主要用diff队列保存需要更新的DOM后，统一批量更新，Vue使用双向指针，一边对比一边更新DOM)

    # 5. 如何解决跨域

    1. CORS(后端实现)
    2. Proxy(配置devServer)
    3. Nginx

    # 6. Vue中为什么组件中data是函数而不是对象？

    1. Vue实例(根实例)定义data可以是对象、也可以是函数，因为根实例是单例，不会造成数据污染的情况
    2. 组件实例对象定义data必须为函数，目的是防止多个组件实例对象之间公用一个data，造成数据污染

    # 7. Vue的双向绑定

    # 原理

    数据劫持结合发布订阅模式，即数据的改变会引发视图更新，反之亦然

    # v-model的原理

    v-model本质是一个语法糖(value + input事件)

    # 8. 生命周期

    # 8.1 Vue2.x的生命周期

    1. 创建前后
    2. 挂载前后
    3. 更新前后
    4. 销毁前后
    5. 激活前后(仅keep-alive组件可用)
    6. errorCaptured(捕获一个来自子孙组件的错误时调用)

    # 具体阶段分析

    beforeCreate -> created

    • 初始化Vue实例，进行数据监测，此时data还未创建

    created

    • 完成数据监测和配置项(watch、computed..)的配置
    • 此时可以访问到data，vm.$el并未创建

    created -> beforeMount

    • 判断是否存在el选项，若不存在则停止编译，直至调用vm.$mount(el)再继续编译
    • render -> template -> outerHTML
    • vm.el获取的是挂载的DOM

    beforeMount

    • 此时可以拿到vm.el，vm.el已经初始化完毕，但未上树

    beforeMount -> mounted

    • vm.el上树完毕
    • vm.$el生成的DOM替换了el对应的DOM

    # 使用场景

    beforeCreate

    • 此时组件实例还未创建，可用于插件开发中执行一些初始化任务

    created

    • 此时组件初始化完毕，data可用，可用于发网络请求
    • 也可以选择在mounted中发请求，但mounted中发请求会造成页面闪动(此时DOM已经生成)

    beforeMount

    • 此时未执行渲染、更新，DOM未创建

    mounted

    • DOM渲染挂载完毕，可用于获取访问数据、DOM元素

    beforeUpdate

    • 此时修改数据，不会触发更新方法，可用于获取更新前的状态

    updated

    • 此时状态是最新的，更新状态会触发更新方法

    beforeDestory

    • 可用于清除定时器、取消订阅等

    # 9. 什么是虚拟DOM

    是对真实DOM的抽象，以VNode为基础的树，使用对象来描述一个节点，最终通过一系列操作将虚拟DOM转为真实DOM

    # 9.1 为什么需要虚拟DOM

    • 操作DOM非常消耗性能，频繁操作会使页面卡顿
    • 虚拟DOM最大的优势是diff算法，减少js操作真实DOM的性能消耗

    # 原生操作DOM和虚拟DOM操作DOM的区别

    1. 原生操作DOM，浏览器会从构建DOM树开始从头到尾执行一遍，如果需要更新10个节点，就需要执行10次
    2. 虚拟DOM不会立即操作DOM，而是将这10次更新的diff内容存到本地一个js对象中，最后一次性映射到DOM上，减少性能消耗

    # 10. v-if和v-show的区别

    • v-if是将元素从DOM上移除或添加，性能开销较大，类似于display: block/none
    • v-show是将元素从视觉上隐藏或显示，类似于visibility: hidden
    • 频繁切换，使用v-show，反之则使用v-if

    # 11. v-if和v-for的优先级是？

    v-for > v-if

    • 永远不要将这两个指令同时用在同一个元素上，会造成性能浪费(每次渲染会先进行循环再判断)

    # 最佳实践

    <template v-if="">
    <span v-for=""></span>
</template>

    # 12. Vue中为什么要在v-for时添加上key

    key是给每一个VNode一个唯一id，是diff的优化策略，diff可以根据key更加快速查找到对应的VNode

    • 当 Vue.js 用 v-for 正在更新已渲染过的元素列表时，它默认用“就地复用”策略
    • 如果数据项的顺序被改变，Vue 将不会移动 DOM 元素来匹配数据项的顺序， 而是简单复用此处每个元素，并且确保它在特定索引下显示已被渲染过的每个元素

    # 13. 说说nextTick

    由于Vue的数据驱动视图更新是异步的，会等待同一事件循环中所有数据更新完毕后，统一进行视图更新

    即Vue是异步DOM更新的，状态发生变化时，Vue会开启异步队列，视图需要等待异步队列中所有状态变化完成后，再统一更新

    原理

    • 借助EventLoop

    目的

    • 性能优化
      • nextTick返回的永远是一个Promise，即微任务，所以会等待同步任务执行完毕以后再去执行微任务

    # 14. watch & watchEffect

    # 14.1 watch

    监听数据的变化

    基本用法: watch(target, (newVal, oldVal) => {}, options)

    target

    • 可以是数组、具体属性、箭头函数的返回值

    配置项：

    • deep
      • 在 ref 嵌套的深层次对象中有效，ref 开启 deep 后，新旧的值是一致的，reactive对象默认开启了 deep
    • immediate
      • 立即执行

    # 14.2 watchEffect

    会立即执行传入的函数(非惰性)，并响应式地追踪其依赖项，并在依赖发生变化时重新执行函数

    基本用法:

    const stopWatch = watchEffect((clearEffect) => {
  console.log(message.value)
  clearEffect(() => {
    console.log('watchEffect执行时，会优先调用该函数，常用于清除副作用')
  })
}, {
  flush: 'pre | sync | post',
  onTrigger (e) {}
})

stopWatch() // 停止监听

    配置项:

    • flush
      • pre：组件更新前执行
      • sync：强制效果始终同步触发
      • post：组件更新后执行，此时在watchEffect中可以拿到DOM元素
    • onTrigger: Function 用于调试
    watchEffect(() => {
  let btn: HTMLButtonElement = document.getElementById('btn') as HTMLButtonElement
  console.log(btn)
}, {
  flush: 'post'
})

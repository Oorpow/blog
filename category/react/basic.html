}

class TestLifeCycle extends Component<{}, IState> {
  state = {
    currentNum: 0,
  }

  render() {
    const { currentNum } = this.state

    return (
      <div>
        <button type="button" onClick={this.increment}>
          click
        </button>
        <span>{currentNum}</span>
      </div>
    )
  }

  increment = () => {
    this.setState(
      (preState) => {
        console.log(preState)
        return {
          currentNum: preState.currentNum + 1,
        }
      },
      () => {
        console.log(this.state)
      }
    )
  }
}

    # 2. 事件系统

    # 2.1 合成事件的绑定

    JSX中，必须使用驼峰形式来书写事件属性名

    <button onClick={this.handleClick}></button>

    # 2.2 合成事件的实现机制

    在React底层，针对合成事件做了：事件委派、自动绑定

    1. 事件委派
      1. React不会将事件处理函数直接绑定到真实DOM上，而是将所有事件绑定到结构最外层，使用一个统一的事件监听器
      2. 该监听器上，维护了一个映射来保存所有组件内部的事件监听和处理函数
      3. 组件挂载或卸载时，只是从监听器上插入或删除一些对象
      4. 事件触发时，先由监听器处理后，再到映射中找到真正的事件处理函数并调用
    2. 自动绑定
      1. React组件中，每个方法的上下文都会指向组件的实例(自动绑定this为当前组件)
      2. 使用ES6的class或纯函数时，自动绑定就失效了，需要手动实现this绑定
        1. bind
        2. 构造器内声明
          1. this.handleClick = this.handleClick.bind(this)
        3. arrow func

    # 3.1 受控组件与非受控组件

    # 3.1.1 受控组件

    数据源于组件的state

    受控组件更新state流程

    • 初始state中，设置表单默认值
    • 表单值发生变化时，触发onChange事件处理器
    • onChange通过合成事件对象e，拿到改变后的状态，并更新state
    • setState触发视图更新，完成表单值更新

    # 3.1.2 非受控组件

    一个表单组件没有value props | checked prop

    组件的默认状态使用：defaultValue | defaultChecked prop

    # 5. HOC

    接收React组件作为输入、输出一个新的React组件

    实现高阶组件的方法如下

    • 属性代理：HOC通过被包裹的React组件来操作props
    • 反向继承：HOC继承于被包裹的React组件

    # 5.1 HOC的功能

    • 控制props(newProps)
    • 通过refs使用引用
    • 抽象state(将原组件抽象为展示组件，分离状态)

    # 5.2 属性代理实现HOC

    interface NewProps {
  currentVal: string
  changeName: Function
}

// 高阶组件MyContainer
const MyContainer = (WrapperComponent: ComponentClass) => 
  class extends Component {
    constructor(props: any) {
      super(props)
    }

    state = {
      name: 'aa',
    }

    changeName = (name: string) => {
      this.setState({ name })
    }

    render() {
      const newProps: NewProps = {
        currentVal: this.state.name,
        changeName: this.changeName,
      }
      // render方法中返回了传入的React组件，因此就可以通过HOC来传递props，这种方法即属性代理
      return <WrapperComponent {...this.props} {...newProps} />
    }
  }

    // 通过decorator调用HOC，接收需要装饰的类作为参数，返回一个新的内部类
@MyContainer
class MyComponent extends Component<any, any> {
  render() {
    return <button type="button" onClick={() => this.props.changeName('haha')}>
        submit
      </button>
  }
}

    # 6. 组件性能优化

    # 6.1 PureRender

    Pure指组件满足纯函数的条件，即组件的渲染是被相同的props、state渲染，进而得到相同结果

    # 6.1.1 PureRender本质

    原理：重新实现了scu方法，通过对新旧props、state浅比较进行性能优化

    要作到充分比较，只能通过深比较，但性能开销大，PureRender对object只做了引用比较，并没有作值比较

    function shallowEqual(obj, newObj) {
  if (obj === newObj) return true

  const objKeys = Object.keys(obj)
  const newObjKeys = Object.keys(newObj)

  if (objKeys.length !== newObjKeys.length) return false

  // 关键代码，只需关注新旧props中每一个是否相等，并不深入对比值
  return objKeys.every(key => newObj[key] === objKeys[key])
}

    传递数据时，直接使用不可变数据来提升组件渲染性能

    React建议将this.state当作不可变的，修改前需要做一个deep_clone

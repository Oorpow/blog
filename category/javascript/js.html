    # BOM的理解、常见的对象

    BOM即浏览器对象模型，主要用于与浏览器的交互，如页面的前进后退、获取浏览器信息等

    常见对象

    • window(浏览器实例)
    • navigator(浏览器属性)
    • location(URL)
    • screen(客户端的显示器信息)
    • history

    # 说说对作用域、作用域链的理解

    js遵循词法作用域，即变量被创建时就已经确定好了

    # 作用域

    变量、函数生效的区域

    分类

    • 全局作用域
    • 函数作用域
    • 块级作用域

    # 作用域链

    JS引擎查找JS变量时，会先在当前作用域查找，若找不到则向上层作用域寻找，直到全局作用域，若仍未找到，非严格模式下会隐式声明该变量或报错

    # 说说执行上下文、执行栈

    执行上下文是对JS代码执行环境的抽象

    # 分类

    • 全局执行上下文
    • 函数执行上下文

    # 生命周期

    • 创建（确定this，创建变量环境、词法环境）
    • 执行（执行变量赋值，执行代码）
    • 回收（执行上下文出栈等待虚拟机回收）

    执行栈

    即调用栈，用于存储在代码运行期间创建的所有上下文

    # 说说闭包、使用场景、优劣

    定义在一个函数内部的函数，该函数可以访问到外层函数的变量、形参

    使用场景

    • 防抖、节流
    • IIFE

    优点

    • 可以创建出私有变量
    • 延长变量的生命周期

    缺点

    • 过多的闭包会导致性能低下
    • 保存在内存中的变量如何长期不释放，会造成内存泄漏

    # 说说事件模型

    1. 原始事件模型(DOM0)
    2. 标准事件模型(DOM2)
    3. IE事件模型

    # DOM/0

    特点

    • 跨浏览器兼容性好
    • 绑定速度快

    缺点

    • 绑定速度过快，可能会出现DOM还没加载完毕，以至于事件无法正常运行
    • 同类型事件只能绑定一次
    • 只支持冒泡

    # DOM/2

    特点

    • 可以绑定多个事件处理器
    • 触发事件经历事件捕获、处于目标、事件冒泡三个阶段

    # IE

    事件流不包含捕获 事件监听与解绑

    attachEvent(eventType, handler)
detachEvent(eventType, handler)

    # 说说事件流

    事件在目标元素和祖先元素之间的触发顺序

    # 事件捕获

    从最不具体的元素向下一直传递到目标元素身上

    # 事件冒泡

    从目标元素一直向上传递到最不具体的元素

    # 说说事件代理

    将一个或一组元素的事件委托给它的父级或外层元素

    # 原理

    借用事件冒泡，当事件响应到目标元素后，会通过事件冒泡传递到外层绑定了事件的元素，然后在外层元素上执行函数

    # 说说事件循环

    JS是单线程语言，同步任务会进行主线程，异步任务会进入消息队列，在同步任务执行完毕后，会去消息队列取出异步任务并推入主线程执行其回调，上述过程不断地重复就形成了事件循环

    # 异步任务

    可以分为微任务、宏任务，执行过程中，微任务优先级会高于宏任务

    # 微任务

    • Promise.then
    • Proxy

    # 宏任务

    • 定时器
    • I/O(nodeJs)
    • UI事件

    # 说说JS本地存储的方式有哪些、区别

    1. localStorage
    2. sessionStorage
    3. cookie

    # localStorage

    • 本地存储
    • 除非手动删除，否则永远存在
    • 存储的信息，同一域中能够共享
    • 只能存入字符串

    # sessionStorage

    • 会话存储
    • 浏览器一关闭就会清除

    # cookie

    • 用于保持用户登录态
    • 大小不超过4k
    • 设置Secure属性，仅在HTTPS协议下发送，可以提高安全性
    • 生命周期由服务器设置或设置Expires、Max-age
    • cookie的数据会自动发给服务器

    # 说说new操作符执行了哪些操作？

    依据给定的构造函数创建出一个它的实例对象

    1. 创建一个新的对象
    2. 将该对象与构造函数的原型对象进行连接
    3. 修改构造函数的this指向
    4. 根据构造函数的返回值进行判断，基本类型忽略，引用类型需要正常处理

    # 说说原型、原型链

    每个实例对象身上都有__proto__属性指向其构造函数的原型对象，其身上也有__proto__属性，以此类推逐层向上，直到一个对象的原型对象为null，这整个过程就构成了原型链，null为原型链的尽头

    原型链充分说明了为什么一个对象身上会有另一个对象的属性或方法

    # 注意

    构造函数属于函数对象，是通过Function构造器生成的

    • Object.prototype.__proto__ === null
    • Object.__proto__ === Function.prototype
    • Person.prototype.__proto__ === Object.prototype

    # == 与 === 的区别

    # ==

    • 允许类型转换
    • 若两个比对的值，任一方是boolean || string，会先转为number再比对
    • 若任一方是引用数据类型，则会调用valueOf()转为原始值再比对
    • 若任一方是NaN，返回false
    • null == undefined null == null undefined == undefined
    • NaN != NaN

    # ===

    • 不允许进行类型转换

    # Tips

    比较null时，使用==，其它情况尽量用===

    if (a == null)
// 等同于
if (a === null || a === undefined)

    # 0.1 + 0.2 === 0.3

    • 进制转换
      • 整数转二进制 除2取余 从后往前排
      • 二进制转十进制 乘2取整 从前往后排
    • 进制转换API
      • (57).toString(2)
      • 二转十, parseInt('二进制', 2)

    结果

    • .1 + .2 !== .3
    • 小数在取二进制数时，会存在余数无法保证严格相等于.3

    # 说说何时发生隐式转换？

    1. 比较运算(<> == ===) 需要用到boolean
    2. 算术运算(+ - * / %) 需要用到number
    3. 用到string

    # boolean

    下列均为false，在需要用到boolean时

    • undefined、null、NaN
    • +0、-0
    • ""
    • false

    # string

    用到string或+时，会进行字符串拼接

    • '1' + true => '1true'
    • '1' + {} => '1[object Object]'
    • '1' + [] => '1'
    • '1' + function() {} => '1function() {}'
    • '1' + undefined || null => '1(undefined || null)'

    # number

    除了+外，其它都会将运算转为number

    • '1' - 1 => 0
    • '1' * [] => 0
    • 'ab' - 1 => NaN
    • null + 1 => 1
    • undefined + 1 => NaN

    # 说说typeof和instanceof的区别

    # 注意

    • typeof null === 'object'，js会返回true，实际是错误的，null本身不是对象
    • 除了function，typeof会返回'function'，其余的引用类型均为object
    • 若需要判断通用类型，可以使用Object.prototype.toString()，返回[object xxx]的字符串

    区别

    • typeof返回string，表示操作符的类型，能够准确判断基本数据类型(null除外)
    • instanceof返回boolean,用于检测构造函数的prototype是否出现在某个实例对象的原型链上，能够准确判断复杂数据类型

    # 什么是ajax，如何实现

    异步的JS和XML，网页开发技术，可以在不重新加载整个页面的情况下，向服务器发请求拿数据，操作DOM进行页面的局部更新

    # 原理

    • XMLHttpRequest

    # 实现

    function myAjax(options) {
    const xhr = XMLHttpRequest()
    options = options || {}
    options.type = (options.type.toUppercase()) || 'GET'
    options.dataType = options.dataType || 'json'

    if (options.type === 'GET') {
        xhr.open('GET', options.url + '?' + options.data, true)
    } else if (options.type === 'POST') {
        xhr.open('POST', options.url, true)
        xhr.send(options.data)
    }
    

    xhr.onreadystatechange = function() {
        if (xhr.readyState === 4) {
            if (xhr.status >= 200 && xhr.status < 300) {
                options.success && options.success(xhr.responseText, xhr.responseXML)
            } else if (xhr.status >= 400) {
                options.fail && options.fail(xhr.status)
            }
        }
    }
}

myAjax({
    url: '',
    method: '',
    data: {},
    dataType: '',
    success() {},
    fail() {}
})

    # 能够改变this指向的方法有哪些、区别、如何实现

    1. bind
    2. call
    3. apply

    # 区别

    • 参数：bind可以分多次传入，call是以参数列表形式传入，apply以数组形式
    • 调用：call和apply都是临时改变一次，并自动调用一次函数，bind是返回一个永久改变了this指向的函数
    Function.prototype.myCall = function(context) {
    if (typeof this !== 'function') return
    context = context || window
    context.fn = this

    let result = context.fn()
    delete context.fn
    return result
}

    Function.prototype.myApply = function(context) {
    if (typeof this !== 'function') return
    let args = arguments
    let result

    context = context || window
    context.fn = this
    if (args[1] && Array.isArray(args[1])) {
        result = context.fn(args[1])
    } else {
        result = context.fn()
    }
    delete context.fn
    return result
}

    Function.prototype.myBind = function(context) {
    if (typeof this !== 'function') return
    let args = [...arguments].slice(1)
    let _this = this
    let emptyFn = function() {}

    context = context || window

    return function Fn() {
        args = args.concat(...arguments)
        return fn.apply(this instanceof Fn ? new fn(...arguments) : args)
    }
}

    # DeepClone & ShallowClone

    # ShallowClone

    创建一个新的数据，该数据有着原始数据的一份精确拷贝

    function shallowClone(target) {
    let obj = target instanceof Array ? [] : {}
    for (let k in target) {
        if (target.hasOwnProperty(k)) {
            obj[k] = target[k]
        }
    }
    return obj
}

    # DeepClone

    深层次的递归拷贝，会创建一个新的对象，对于基本数据类型直接拷贝值，引用类型的数据则新开堆栈，创建出新的内存空间存放数据

    function deepClone(target) {
    let obj = target instanceof Array ? [] : {}
    for (let k in target) {
        if (target.hasOwnProperty(k)) {
            if (target[k] && typeof target[k] === 'object') {
                obj[k] = deepClone(target[k])
            } else {
                obj[k] = target[k]
            }
        }
    }
    return obj
}

    # JS中的继承的优点、有哪些继承方式？如何实现？

    优点

    1. 可以使子类拥有父类的属性、方法
    2. 子类可以重写父类的方法、拥有自己的特点

    分类

    1. 构造函数继承
    2. 原型继承
    3. 组合式继承
    4. 寄生组合式继承（最佳实践）

    # 构造函数继承

    痛点

    • 无法继承父类原型上的方法
    function Dad(name) {
    this.name = name
}
function Son(name, age) {
    Dad.call(this, name)
    this.age = age
}

    # 原型继承

    痛点

    • 多个实例共享原型对象，修改一个会导致全改
    function Dad(name) {
    this.name = name
}
function Son(age) {
    this.age = age
}
Dad.prototype.sayName = function() { return this.name }
Son.prototype = new Dad()

    # 组合式继承

    痛点

    • 父类构造方法调用了多次，造成性能上的浪费
    function Dad(name) {
    this.name = name
}
function Son(name, age) {
    Dad.call(name)
    this.age = age
}
Dad.prototype.sayName = function() { return this.name }
Son.prototype = new Dad()
Son.prototype.constructor = Son

    # 寄生组合式继承

    function Dad(name) {
    this.name = name
}
function Son(name, age) {
    Dad.call(name)
    this.age = age
}
Dad.prototype.sayName = function() { return this.name }
Son.prototype = Object.create(Dad.prototype)
Son.prototype.constructor = Son

    # 说说防抖和节流、使用场景、如何实现？

    # 防抖

    触发事件后，若在一段时间重复触发则重新计时

    function debounce(fn, wait, imme) {
    let timer
    return function() {
        if (timer) {
            clearTimeout(timer)
        }
        if (imme) {
            let callNow = !timer
            timer = setTimeout(() => {
                timer = null
            }, wait)
            if (callNow) {
                fn.apply(this, arguments)    
            }
        } else {
            timer = setTimeout(() => {
                fn.apply(this, arguments)
            }, wait)
        }
    }
}

    # 节流

    触发事件后，若在一段时间后重复触发仅生效一次

    function throttle(fn, wait) {
    let lastTime = Date.now()
    let timer
    return function() {
        let currentTime = Date.now()
        let surplusTime = wait - (lastTime - currentTime)
        if (surplusTime <= wait) {
            fn.apply(this, arguments)
            lastTime = Date.now()
        } else {
            timer = setTimeout(fn, surplusTime)
        }
    }
}

    # 如何判断元素是否出现在页面的可视区域中

    公式: 元素在页面中的位置 - 滚动条滚动的距离 <= 页面可视区域

    实现

    function isDisplay(el) {
    let viewportHeight = document.documentElement.clientHeight || window.innerHeight || document.body.clientHeight
    let scrollTop = document.documentElement.scrollTop
    let elPos = el.offsetTop
    return viewportHeight >= elPos - scrollTop
}
